# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Static personal portfolio + blog for Aditya Bankoti, deployed to GitHub Pages. No build step — raw HTML/CSS/JS served directly. Push to `master` auto-deploys via `.github/workflows/static.yml`.

## File structure

```
index.html              Portfolio (single page)
styles.css              Portfolio styles
script.js               Portfolio JS (nav toggle, scroll effect)
blog/
  index.html            Blog listing (posts rendered from JS array)
  blog.css              Shared blog styles (nav + listing + prose)
  posts/
    _template.html      Copy this to write a new blog post
    *.html              Individual blog posts
output_web/             Pixelated PNG hero images (generated by script.py)
script.py               Image processing pipeline (see below)
```

## Architecture

### Portfolio (`index.html` + `styles.css`)
Single-page design with: hero, projects (collapsible accordion by category), experience (timeline), education (timeline), hobbies (grid), contact. No external CSS dependencies — pure custom CSS with CSS variables. Google Fonts (Inter + JetBrains Mono) loaded via CDN.

**Design tokens** are all in `:root` in `styles.css` — `--bg`, `--accent`, `--font-mono`, etc. Changing the color scheme is a one-file edit.

### Blog system
- **Listing** (`blog/index.html`): Posts are a JS array (`POSTS = [...]`) at the top of the file. Tag filtering and card rendering happen client-side. To add a post, push an entry to the array.
- **Post pages** (`blog/posts/*.html`): Each post is a standalone HTML file. They load CDN libraries for features:
  - **Prism.js** (syntax highlighting): `prism-tomorrow` theme + autoloader plugin. Use `<pre><code class="language-NAME">` for code blocks.
  - **KaTeX** (LaTeX math): Auto-render extension. Use `$...$` for inline math, `$$...$$` for display math.
  - **Copy buttons**: added by inline JS at page bottom.
- **Template** (`blog/posts/_template.html`): Fully commented with usage examples for code, math, and callout boxes.

### Adding a new blog post
1. `cp blog/posts/_template.html blog/posts/your-slug.html`
2. Fill in title, date, tags, read time, and content
3. Add entry to `POSTS` array in `blog/index.html`
4. Push to `master`

### Relative paths
- From `blog/index.html`: portfolio root is `../`
- From `blog/posts/*.html`: portfolio root is `../../`, blog index is `../`

## Background image pipeline

`script.py` converts photos into pixelated PNGs for the hero slideshow (now unused in the default portfolio design but kept for reference):
```bash
# Place source images in input_files/ (gitignored), then:
python script.py
# Output lands in output_web/
```
Requires `opencv-python`, `Pillow`, `pillow-heif` in a venv (e.g. `my-venv/`, gitignored).
